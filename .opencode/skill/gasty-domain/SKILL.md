# Gasty — Domain Model & Conventions

Gasty is a personal expense tracker, es-AR locale, mobile-first PWA. The data is the product. This skill is the source of truth for the domain shapes and the end-to-end flow.

> **Fuente única de tipos.** `Transaction`, `Category`, `RecurringConfig`, `Settings`, `ParsedTransaction`,
> `Tab` se definen acá (y en `src/types/index.ts`). `gasty-data-layer` referencia estos tipos para el
> schema Dexie; no los dupliques. El flujo e2e de abajo también se menciona en `gasty` (Canon).

## Core types (`src/types/index.ts`)

```ts
export type TransactionType = 'expense' | 'income'

export type RecurringKind = 'fixed' | 'fixed_temporary' | 'none'

export interface RecurringConfig {
  kind: RecurringKind
  totalMonths?: number     // only for 'fixed_temporary'
  currentMonth?: number    // only for 'fixed_temporary'; starts at 1
  invoiceDay?: number      // day-of-month the recurring charges (1-31)
}

export interface Transaction {
  id: string               // crypto.randomUUID()
  type: TransactionType
  amount: number           // positive number; sign is derived from `type`
  description: string
  categoryId: string       // FK to Category.id
  date: string             // 'YYYY-MM-DD' local ISO, NEVER UTC
  recurring: RecurringConfig
  originalId?: string      // present ONLY for clones generated by checkAndCloneRecurring
  createdAt: string        // full ISO timestamp
}

export type CategoryType = 'expense' | 'income' | 'both'

export interface Category {
  id: string               // stable, snake_case; never rename
  name: string             // Spanish display name
  emoji: string            // single emoji
  color: string            // hex with '#', e.g. '#f59e0b'
  type: CategoryType
}

export type Theme = 'light' | 'dark'
export type Currency = 'ARS' | 'USD'

export interface Settings {
  theme: Theme
  currency: Currency
}

export interface ParsedTransaction {
  type: TransactionType
  amount: number
  description: string
  categoryId: string
  date: string
  recurring: RecurringConfig
}

export type Tab = 'dashboard' | 'transactions' | 'stats' | 'settings'
```

## Defaults

- `theme`: `'light'`
- `currency`: `'ARS'`
- `seedDatabase()` writes these defaults on first run; never overwrite on subsequent boots.
- `SETTINGS_ID = 'app-settings'` (constant in `db.ts`).

## End-to-end flow

```
User types in SmartInputSheet
        │
        ▼
parseInput(text)        → ParsedTransaction | null
        │
        ▼
createTransactionFromParsed(parsed)  → Transaction (with id, createdAt)
        │
        ▼
db.transactions.add(tx)  or  db.transactions.put(tx)  (edit)
        │
        ▼
Dexie change fires
        │
        ▼
useLiveQuery subscribers re-render (Dashboard, Transactions, Stats, Settings)
        │
        ▼
UI updates. No manual refetch, no useEffect chain.
```

On app mount, `useRecurringCheck()` calls `checkAndCloneRecurring()` exactly once. It iterates over all source transactions (those with `recurring.kind !== 'none'` and no `originalId`) and creates a clone for the current month if one doesn't exist yet. Clones carry `originalId` and an incremented `currentMonth`.

## Money & dates (use the helpers, never `toLocaleString` directly)

`src/lib/format.ts`:

| Helper | Output |
|---|---|
| `formatMoney(amount, currency)` | `$ 1.500`, `US$ 15.00` |
| `formatCompact(amount, currency)` | `$ 1,5K`, `$ 2,3M` |
| `formatDate(iso)` | `Hoy` / `Ayer` / `Mañana` / `Hace 3 días` / `En 5 días` / `15 jun` |
| `formatDateFull(iso)` | `15 de junio` |
| `formatMonth(date)` | `Junio 2026` |
| `MONTHS_ES`, `MONTHS_FULL` | short and long month names in es-AR |

ARS uses thousands `.` and no decimals. USD uses thousands `,` and 2 decimals. The locale is hardcoded to `es-AR` in the formatters.

## Common mistakes to avoid

- 🟥 Using `toISOString()` for the `date` field — produces off-by-one days in negative timezones. Use `toLocalISO` (defined locally in `parser.ts` and `recurring.ts`).
- 🟥 Storing `amount` as a string or with a sign — it's always positive; the sign is derived from `type`.
- 🟥 Editing a clone directly — always edit/delete the source, let the recurring hook regenerate.
- 🟥 Looking up a category by name — use the stable `id`.
- 🟥 Mixing month key formats — always `'YYYY-MM'` (10 chars) derived from local ISO date.
